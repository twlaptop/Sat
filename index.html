재귀 피보나치는 중복 계산이 많아 O(2ⁿ) 시간 복잡도를 가집니다. 세 가지 방법으로 개선할 수 있어요:

1. 메모이제이션 — 이미 계산한 값을 저장해 O(n)으로 줄입니다.
2. 반복문(Bottom-up DP) — 스택 오버플로우 없이 O(n) 시간, O(1) 공간.
3. 행렬 거듭제곱 — O(log n)으로 대규모 n에서 가장 빠릅니다.

어떤 방향이 더 맞는지 알려주시면 코드도 작성해 드릴게요.

물론이죠. functools.lru_cache를 쓰면 가장 간결합니다:

from functools import lru_cache @lru_cache(maxsize=None) def fib(n: int) -> int: if n < 2: return n return fib(n - 1) + fib(n - 2)
fib(1000)도 즉시 반환합니다. 데코레이터 없이 직접 딕셔너리로 관리하는 버전도 필요하면 말씀해 주세요.

Anthropic은 AI의 안전성 연구를 핵심 미션으로 삼는 회사입니다. Claude는 그 연구의 산물이자, 실제 세상에서 안전하고 유익한 AI가 가능하다는 증거입니다.

Claude는 모르는 것을 모른다고 말하고, 불확실한 것을 불확실하다고 말합니다. 정답보다 정직함이 우선입니다.

  • 헌법적 AI(Constitutional AI)로 가치 기반 훈련
  • 인간 피드백을 통한 지속적 개선(RLHF)
  • 편향과 환각을 줄이기 위한 적극적 연구
  • 사용자 프라이버시와 데이터 보호 최우선
