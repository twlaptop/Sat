> 마지막 업데이트: 2026-07-10 (오후)

---

## 이 시스템이 뭔가

삼성전자·SK하이닉스·하나마이크론 등 협력사 현장에 파견되는 엑스티 근로자들의
**출입 시간을 버튼 하나로 기록하고 관리하는 서버**입니다.

```
근로자가 앱에서 버튼 1번
       ↓
이 서버에 입실/퇴실 시각 저장
       ↓
관리자가 웹에서 기록 조회 + 엑셀 출력
```

| **D파트 (관리자 웹)** | 관리자가 쓰는 웹 화면 | 🔄 제작 중 |

## 로그인 흐름

### 최초 1회 — 계정 활성화
```
앱 최초 설치
    ↓
주민번호 입력
    ↓
서버에서 암호화된 주민번호와 대조 (본인 확인)
    ↓
일치하면 계정 활성화 완료
    ↓
이후부터는 주민번호 불필요
```
> 주민번호는 최초 1회만 사용하며 서버에 암호화(bcrypt 해시)된 형태로만 보관됩니다.
> 원본 주민번호는 서버에 저장되지 않습니다.

### 매번 로그인
```
이름 + 생년월일 6자리 입력 (예: 950110)
    ↓
조직도에 없는 사람은 자동 차단
    ↓
로그인 성공 → 인증 토큰 발급 (유효시간: 30분)
```

## 용어 설명

| 용어 | 뜻 |
|---|---|
| 입실 | 라인(공정)에 들어가는 것 |
| 퇴실 | 라인(공정)에서 나오는 것 |
| 이석 | 잠깐 라인 밖에 나가는 것 (PM외부대기·면담 등) |
| 회차 | 하루에 여러 번 입실할 때 몇 번째인지 (1회차, 2회차…) |
| 수정 요청 | 입실/퇴실 깜빡하거나 잘못 눌렀을 때 수정 신청하는 것 |
| 역할 구분 | worker(본인 기록만) / leader(팀 조회) / admin(전체+수정 승인+계약시간 관리) |
| 최종관리자 | admin 중 최상위 권한자. is_final_admin=True로 구분, 최대 3명 |
| 지각 판정 | 입실 시각이 근무표 기준시각 초과 시 자동 표시 (D/주=06:00, S/야=18:00, O=08:00) |
| 계약시간 | 사업장+공정+라인 조합별 정해진 작업 시간. 퇴실 알람 기준으로 사용 |
| API | 앱·웹이 이 서버와 통신하는 방식 |
| 마이그레이션 | DB 구조를 바꾸는 작업 (이력 보존됨) |
| 엑스티톡 | 작업자가 관리자에게 문의하는 앱 내 채팅 기능 |
| bcrypt | 주민번호 등 민감 정보를 암호화하는 방식. 원본 복원 불가 |
