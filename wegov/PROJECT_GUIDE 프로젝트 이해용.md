> 마지막 업데이트: 2026-07-11

---

## 이 시스템이 뭔가

삼성전자·SK하이닉스·하나마이크론 등 협력사 현장에 파견되는 엑스티 근로자들의
**출입 시간을 버튼 하나로 기록하고 관리하는 서버**입니다.

```
근로자가 앱에서 버튼 1번
       ↓
이 서버에 입실/퇴실 시각 저장
       ↓
관리자가 웹에서 기록 조회 + 엑셀 출력
```

## 핵심 설계 결정

> 개발자가 코드 수정 시 반드시 알아야 할 규칙들입니다.

### 인증 구조
- 로그인 → Access Token (2시간) + Refresh Token (30일) 동시 발급
- 로그인 응답에 `role`, `name`, `worker_id` 포함 — 추가 API 호출 불필요
- 퇴직 처리 또는 역할 변경 시 → `token_invalidated_at` 갱신 → 기존 토큰 즉시 차단
- 모든 인증 필요 API는 `get_current_worker` 사용 — is_active + token_invalidated_at 동시 체크

### 역할 체계
| 역할 | 값 | 할 수 있는 것 |
|---|---|---|
| 작업자 | worker | 본인 기록만 |
| 리더·인사팀 | leader | 팀 전체 조회·통계·수정 대기 목록 |
| 관리자 | admin | 전체 관리·승인·등록·삭제 |
| 최종관리자 | admin + is_final_admin=True | admin 중 최상위 (최대 3명) |

### 데이터 보존 원칙
- Worker 물리 삭제 금지 → `is_active=False`로 비활성화
- WorkRecord 물리 삭제 금지 → `is_voided=True`로 무효 표시
- 수정 승인 시 원본 기록 보존, 새 기록 또는 시각 수정

### 입실 흐름
```
앱 실행
↓
GET /checkin/today-schedule → 오늘 스케줄 있으면 자동 표시
↓
스케줄 있음: POST /checkin/from-schedule
스케줄 없음: GET /sites → GET /processes?site_name= → POST /checkin/manual
↓
POST /checkout (퇴실)
```

### 퇴실 알람 흐름
```
입실 직후
↓
GET /process-contracts/lookup?site_name=&process=&line=
↓
응답의 contract_hours × 60 = 알람까지 남은 분수
↓
앱에서 타이머 설정
```

## 로그인 흐름

### 최초 1회 — 계정 활성화
```
앱 최초 설치
    ↓
POST /auth/activate (이름 + 주민번호)
    ↓
서버에서 암호화된 주민번호와 대조 (본인 확인)
    ↓
일치하면 계정 활성화 완료 → Access Token + Refresh Token 발급
    ↓
이후부터는 주민번호 불필요
```
> 주민번호는 최초 1회만 사용하며 서버에 암호화(bcrypt 해시)된 형태로만 보관됩니다.

### 매번 로그인
```
POST /auth/login (이름 + 생년월일 6자리)
    ↓
로그인 성공 → Access Token(2시간) + Refresh Token(30일) 발급
응답에 role, name, worker_id 포함
```

### 자동 로그인 (앱 재실행 시)
```
저장된 Refresh Token 사용
    ↓
POST /auth/refresh
    ↓
새 Access Token + 새 Refresh Token 발급
```

## 용어 설명

| 용어 | 뜻 |
|---|---|
| 입실 | 라인(공정)에 들어가는 것 |
| 퇴실 | 라인(공정)에서 나오는 것 |
| 이석 | 잠깐 라인 밖에 나가는 것 (PM외부대기·면담 등) |
| 회차 | 하루에 여러 번 입실할 때 몇 번째인지 (1회차, 2회차…) |
| 수정 요청 | 입실/퇴실 깜빡하거나 잘못 눌렀을 때 수정 신청하는 것 |
| 역할 구분 | worker(본인 기록만) / leader(팀 조회) / admin(전체+수정 승인+계약시간 관리) |
| 최종관리자 | admin 중 최상위 권한자. is_final_admin=True로 구분, 최대 3명 |
| 지각 판정 | 입실 시각이 근무표 기준시각 초과 시 자동 표시 (D/주=06:00, S/야=18:00, O=08:00) |
| 계약시간 | 사업장+공정+라인 조합별 정해진 작업 시간. 퇴실 알람 기준으로 사용 |
| Access Token | 로그인 후 받는 인증 열쇠 (2시간 유효) |
| Refresh Token | 자동 로그인용 토큰 (30일 유효) — 만료 없이 앱 유지 가능 |
| token_invalidated_at | 이 시각 이전에 발급된 토큰은 즉시 차단 — 퇴직·역할 변경 시 갱신 |
| API | 앱·웹이 이 서버와 통신하는 방식 |
| 마이그레이션 | DB 구조를 바꾸는 작업 (이력 보존됨) |
| 엑스티톡 | 작업자가 관리자에게 문의하는 앱 내 채팅 기능 |
| bcrypt | 주민번호 등 민감 정보를 암호화하는 방식. 원본 복원 불가 |
